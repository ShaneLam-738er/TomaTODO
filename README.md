# TomaTODO

一个极简番茄钟网页应用。
打开即用，专注当前这一轮，同时把今天完成了什么安静地记录下来。

## 特点

* **极简计时器**：默认 25 分钟专注，结束后进入休息。
* **本次计划**：开始前写下这一轮要做什么。
* **意外关闭恢复**：不小心关掉页面后，再次打开可以恢复未完成的番茄钟。
* **今日记录栏**：页面右侧显示今天完成的内容，包括开始时间、结束时间和任务说明。
* **今日 / 本周统计**：显示今日完成轮数、今日专注分钟和本周专注分钟。
* **可自定义时长**：支持设置专注、短休、长休和长休间隔。
* **本地存储**：所有数据保存在浏览器 localStorage 中，不需要账号，也不会上传到服务器。
* **导出记录**：可以将番茄钟记录导出为 TXT 文件。

## 使用方式

直接打开网页即可使用。

1. 在输入框里写下本次计划。
2. 点击「开始」。
3. 专注完成后，记录会自动出现在右侧「今日」栏里。
4. 需要调整时间时，打开「设置 / 导出」。

## 数据保存说明

TomaTODO 使用浏览器的 `localStorage` 保存数据。

主要保存内容包括：

* `pomodoro_logs`：已完成的番茄钟记录。
* `pomodoro_settings`：用户自定义的计时时长设置。
* `pomodoro_plan_draft`：当前输入框里的计划草稿。
* `pomodoro_current_session`：当前未完成的番茄钟状态，用于意外关闭后的恢复。

因此，只要你使用的是同一个浏览器、同一个设备、同一个网页地址，替换或更新 `index.html` 通常不会清空旧记录。

需要注意的是，以下情况可能导致记录丢失：

* 更换浏览器或设备。
* 更换网站域名或访问路径。
* 清除了浏览器站点数据。
* 点击了应用内的「清空记录」。

## 意外关闭恢复

如果番茄钟进行到一半时页面被关闭，应用会保存当前剩余时间和任务内容。

再次打开页面后，会提示恢复上次未完成的番茄钟。点击「开始」即可继续。

恢复后不会自动开始计时，这是为了避免打开页面后误启动计时。

## 今日记录栏

右侧极简竖栏用于展示今天已经完成的内容。

每条记录包含：

```text
开始时间-结束时间
任务内容
```

例如：

```text
09:10-09:35
写模电作业第 3 题
```

这个区域只负责回顾今天做了什么，不承担复杂任务管理功能，避免影响主计时器的简洁性。

## 部署到 GitHub Pages

如果你想通过 GitHub Pages 使用：

1. 将 `index.html` 放在仓库根目录。
2. 打开仓库的 Settings。
3. 找到 Pages。
4. 选择部署分支，例如 `main`。
5. 保存后等待 GitHub Pages 构建完成。

之后即可通过 GitHub Pages 地址访问。

## 本地运行

本项目没有构建流程，也不需要安装依赖。

直接双击 `index.html` 打开即可。

也可以用任意静态服务器运行，例如：

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

## 导出记录

在「设置 / 导出」中点击「导出 TXT」，可以下载当前浏览器中保存的番茄钟记录。

导出的格式类似：

```text
[2026-05-08 09:10-09:35] 25 分钟 - 写模电作业第 3 题
```

## 项目定位

TomaTODO 不是复杂的任务管理软件，也不是团队协作工具。

它只解决一个问题：

> 开始一轮专注，完成它，然后看见自己今天确实做了些什么。

所以功能会尽量保持克制：简单、稳定、低打扰。
